# 완전 상용 건축구조해석 솔버 전환 갭 원장

- 기준일: 2026-05-31
- 범위: 기존 워크스테이션 납품 서비스/engineer-in-loop 보조툴을 넘어, **독립 상용 건축구조해석 솔버 제품**으로 주장하기 전에 닫아야 할 개선점과 문서상 미폐쇄 항목 정리
- 비범위: 구현 순서, 담당, 일정, 커밋 단위 계획. 이 문서는 구현 계획이 아니라 **제품 갭 원장**이다.

## 결론

현재 제품은 **워크스테이션 기반 구조해석/최적화 납품 서비스**와 **engineer-in-loop 상용 보조툴**로는 상당히 닫혀 있다. `check_workstation_delivery_readiness.py --json` 기준 `PASS | gates=8/8`, delivery bundle도 `ready`다.

그러나 **완전한 독립 상용 건축구조해석 솔버**로는 아직 닫히지 않았다. `check_independent_product_readiness.py --json` 기준 readiness는 `80/100, blocked`이고, `report_commercialization_level.py --closure-mode strict --json` 기준도 `L3 engineer_in_loop_commercial_assist_ready`, `full_commercial_replacement_ready=false`다.

핵심 차이는 다음이다.

| 구분 | 현재 말할 수 있는 것 | 아직 말하면 안 되는 것 |
| --- | --- | --- |
| 제품 claim | 구조 엔지니어 검토 전제의 상용 해석/최적화 보조 및 납품 서비스 | 독립 상용 구조해석제품, 기술사 검토 대체, 인허가 자동 승인 |
| 솔버 claim | 일부 축약/부분 메쉬/상용 export proxy 기반 검증이 붙은 해석 파이프라인 | 임의 건축 구조에 대한 full 3D nonlinear global FEA solver |
| 검증 claim | 내부 evidence, commercial export sample, model-derived 비교, workstation package smoke | 외부 benchmark receipt 4/4, strict residual holdout closure 3/3, live licensed solver replay |

## 상용 독립 솔버로 가기 위한 개선 작업 원장

### G1. Full 3D Global FEA Core

현재 상태:

- MGT roundtrip/mesh contract는 strong하다.
- `solve_mgt_beam_mesh_3d_global.py`는 real section/material을 일부 주입하고 대표 connected component Newton 수렴 evidence를 낸다.

This rules out simple single-level Galerkin-coordinate reconditioning as the missing G9 lever; the next operator must change hierarchy/interface coupling quality, not just rescale the same subspace.
