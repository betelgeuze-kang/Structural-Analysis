### 1.2 개선 방향

#### 1.2.1 Physics-Informed Hybrid Residual Correction

AI 보정이 물리적 consistency를 항상 만족하도록 constraint를 걸어서 학습 및 실행한다.

- **Global Equilibrium Preserving Correction**
  - AI 보정이 `K(u)u = f` equilibrium을 항상 만족하도록 hard constraint를 걸어서 학습
  - 보정 후 `||K(u)u - f||` residual이 tolerance 내에 들어오지 않으면 자동 폐기

- **Energy Monotonicity Gate**
  - 보정 전후 strain energy가 비감소(non-decreasing) 조건을 만족하는지 검증
  - energy 감소가 감지되면 `blocked` 상태로 전환하고 engineer review queue로 이동

- **Boundary Condition Consistency**
  - 보정된 solution이 기존 support mask, elastic link, story eccentricity를 유지하는지 hard constraint로 검증
  - `mgt_boundary_spring_tangent_receipt.json` 기준 boundary entity violation 시 자동 차단

#### 1.2.2 Adaptive Micro-Continuation + Secant Predictor 개선

현재 `0.656` frontier에서 실패하는 근본 원인은 consistent tangent(Jacobian) 부족이다. 이를 개선하기 위한 방향:

- **Finite-Difference Residual-Jacobian Probe 체계화**
  - bracket 실패 지점(`0.5078125` 등)에서 direct residual `K(u)u - f`를 재평가
  - 4회 accepted correction으로 `10310 N` -> `2092 N`까지 낮춘 경험을 자동화
  - direct residual이 `0.001 N` tolerance 대비 충분히 작아질 때까지 iteration 반복

- **Constrained Anderson Acceleration 파라미터 튜닝**
  - 48-iteration constrained Anderson과 L1-bounded Anderson 결과를 비교하여 자동 선택하는 meta-heuristic 개발
  - residual-trust-region Anderson: 후보를 다시 P-Delta operator에 remap하고 gate 재검사

- **Checkpoint/Resume + Seed Alpha-Scan 자동화**
  - `mgt-uncoarsened-boundary-pdelta-checkpoint.v1` 기반 accepted-state checkpoint/resume을 pipeline으로 자동화
  - relaxation `0.5` segmented continuation과 seed alpha-scan globalization을 자동으로 조합

#### 1.2.3 Active Learning Loop for Residual Correction

solver가 실패하거나 uncertainty가 큰 모델을 자동으로 학습 큐에 넣는 체계를 구축한다.

- **Solver Failure Case 자동 수집**
  - Newton 루프 수렴 실패, fixed-point increment gate 실패, energy monotonicity 실패 케이스를 자동으로 수집
  - 수집된 케이스를 residual correction model의 학습 데이터로 활용

- **OOD (Out-of-Distribution) Auto-Block**
  - uncertainty가 높은 영역에서는 AI correction을 차단하고 `solver-only` 또는 `engineer-review-required` 상태로 자동 전환
  - `ml_surrogate_production_gate.py`의 OOD gate와 연동

- **Uncertainty-Guided Sampling**
  - surrogate model의 uncertainty가 높은 입력 영역을 우선적으로 solver로 평가하여 학습 데이터 다양성 확보

#### 1.2.4 Differentiable Solver / Adjoint 연결 (장기 연구)

- gradient-based design loop를 위한 differentiable solver 개발
- 현재는 제품 claim에 쓸 수 없는 상태이므로 **research track으로 분리**하여 검증

## 5. 핵심 결론

> **"AI가 구조안전성을 자동 보장" 또는 "기술사 검토를 대체"하는 방향이 아니라, "엔지니어 검토 전제의 AI-assisted 설계 대안 탐색" 범위 내에서 고도화해야 한다.**
