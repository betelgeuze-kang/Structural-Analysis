# 건축구조해석 AI 엔진 갭 원장

- 기준일: 2026-05-31
- 범위: 건축구조해석 제품 안에서 “AI 엔진”이라고 부를 수 있는 입력 이해, 해석 surrogate, 최적화 정책, 코드 검토 보조, 설명/감사/MLOps의 부족 영역 정리
- 비범위: 구현 순서, 일정, 담당, 커밋 계획. 이 문서는 **구현 계획이 아니라 AI 제품 갭 원장**이다.
- Claim 경계: 본 문서의 어떤 항목도 구조기술사 검토 대체, 인허가 자동 승인, 완전 자율 설계 확정을 의미하지 않는다.

## 결론

현재 제품의 AI/최적화 계층은 **구조 엔지니어 검토 전제의 결정론적 규칙 기반 탐색 보조**로는 의미 있는 상태다. 그러나 **학습형 AI가 구조해석을 예측하고, 최적 설계를 제안하며, 그 제안이 자동으로 상용 설계 결론으로 승격되는 AI 엔진**으로는 아직 닫히지 않았다.

가장 중요한 사실은 다음이다.

| 구분 | 현재 말할 수 있는 것 | 아직 말하면 안 되는 것 |
| --- | --- | --- |
| AI 최적화 | 규칙 기반/greedy 탐색, solver/proxy gate, replay evidence를 가진 engineer-in-loop 최적화 보조 | 검증된 ML/RL 기반 자율 최적설계 엔진 |
| ML surrogate | opt-in gate와 checkpoint 계약은 있음 | production surrogate가 실제 비용/응답을 바꾸는 상태 |
| 다목적 최적화 | research Pareto archive는 있음 | production Pareto/NSGA/RL policy가 실제 러너에 연결됨 |
| AI 해석 | 연구 스크립트와 잔차학습/메타학습 방향은 있음 | full 3D solver를 대체하거나 보정하는 검증된 AI 해석기 |
| AI 검토 | evidence와 provenance 표시는 강해지는 중 | 법규 조항을 자율 해석해 승인 가능한 설계 판단을 내림 |

## AI 엔진 부족 영역

아래 항목은 기능 구현 순서가 아니라, “상용 건축구조해석 AI 엔진”이라는 claim을 하려면 닫혀야 하는 능력 영역이다.

### AI-G1. 입력 모델 이해와 의미 정규화

현재 상태:

- MGT roundtrip, mesh contract, provenance receipt는 강해지고 있다.
- 현재 bridge MGT lane에서는 `*THICKNESS` row가 PLATE property id로 승격되어 surface smoke solve의 source thickness coverage가 `100%`다.
- 같은 bridge MGT에서 `*OFFSET` beam end offset row `708`개가 typed runtime metadata로 파싱되고, element reference mismatch는 `0`으로 확인되었다. GLOBAL offset은 6DOF frame tangent에도 rigid-end transformation으로 소비되지만, AI가 offset/eccentricity를 추론한 evidence는 아니다.
- 같은 bridge MGT에서 `*CONSTRAINT` support row `8`개, support node `2133`개, `*ELASTICLINK` GEN row `1692`개, `*STORY-ECCEN` setting이 typed boundary entity로 파싱되고 node reference mismatch는 `0`으로 확인되었다. roundtrip rigid-link coarsening은 적용되지만, AI가 boundary condition을 추론하거나 solver가 authored support mask/finite link/story eccentricity를 모두 소비했다는 evidence는 아니다.
- `mgt_boundary_spring_tangent_receipt.json`은 이 deterministic boundary entity를 6DOF sparse boundary subsystem tangent로 소비한다: finite spring component `10152`, restrained DOF `7707`, probe solve relative residual `0.0`. 이는 AI inference evidence가 아니라 typed parser output의 numerical-consumption evidence다.
- `mgt_uncoarsened_boundary_global_equilibrium.json`은 rigid-link coarsening을 끈 deterministic MGT parse에서 authored support mask와 finite elastic-link spring을 full coupled frame-shell global tangent에 직접 조립해 relative residual `1.32e-08`로 solve한다. 이는 AI 추론 evidence가 아니라 source MGT boundary fields의 solver-consumption evidence다.
- `mgt_coarsened_authored_support_pdelta_probe.json`은 coarsened line-only P-Delta 경로에 source MGT authored support mask를 매핑해 본 boundary-condition migration 진단이다. Support node `2133`개 중 coarsened line mesh에 `756`개만 남고, `*ELASTICLINK` `1692` rows 중 양끝 endpoint가 모두 남은 row는 `0`개다. 이 상태로 component-base auto-restraint를 대체하면 첫 load `0.5`부터 fixed-point relative increment `0.752`, max translation `25.79m`로 실패한다. 이는 AI가 boundary를 추론해 고쳤다는 evidence가 아니며, nonlinear path에는 uncoarsened boundary global tangent 연결이 필요함을 보여준다.
- `mgt_uncoarsened_boundary_pdelta_probe.json`은 그 uncoarsened boundary global tangent를 deformed-state P-Delta loop에 실제 연결한다. Authored support `7707` DOF와 finite elastic-link spring component `10152`개를 유지한 채 relaxation `1.0`과 더 촘촘한 `0.35 -> 0.45` micro load steps로 `0.45` load state까지 accepted frontier를 확장했다. `0.45`의 best residual은 `3.08e-05 N`, fixed-point relative increment는 `6.20e-05`, max translation은 `0.642m`다. main probe의 다음 narrow step `0.455`는 residual `2.47e-05 N`으로 residual gate를 통과해도 fixed-point relative increment가 `2.18e-04`로 남아 실패한다. Accepted-state checkpoint/resume tooling도 붙어 `mgt-uncoarsened-boundary-pdelta-checkpoint.v1` displacement NPZ를 node id와 함께 검증하고 재개할 수 있으며, 공식 `mgt_uncoarsened_boundary_pdelta_checkpoint_continuation.json`은 durable `0.45` checkpoint에서 relaxation `0.5` segmented continuation을 이어 `0.5406`까지 accepted frontier를 확장했다. 그 위에서는 seed alpha-scan globalization이 fixed-point correction 방향의 alpha 후보를 다시 P-Delta operator에 remap하고, 필요한 경우 짧은 relaxed 반복을 이어 gate를 재검사한다. 이 경로로 accepted frontier는 `0.656`까지 확장됐고, `0.656`의 residual은 `1.10e-04 N`, fixed-point increment는 `5.18e-06`, max translation은 `0.900m`다. 공식 aggregate의 first failed load는 이제 `null`이지만 full-load `1.0`에는 아직 미달한다. 이는 boundary-condition AI repair 또는 surrogate가 partial nonlinear frontier evidence를 full solver closure로 자동 승인하면 안 된다는 guard evidence이면서, full-load/material Newton closure에는 아직 consistent tangent가 필요함을 보여준다.
- `mgt_direct_residual_newton_probe.json`은 `R(u, λ)=F_int(u)-λF_ext`를 직접 정의해 이 guard를 더 강하게 만든다. 같은 `0.656` checkpoint에서 fixed-point receipt residual은 `1.10e-04 N`이지만, solver regularization 없이 frame service material tangent, frame geometric delta, shell/surface tangent, authored support/finite springs와 reference-configuration `F_ext`를 같은 residual 체계에 넣으면 direct residual은 `4.28e4 N`이다. current tangent Newton correction에 방향성 finite-difference residual-Jacobian alpha 후보를 섞은 trust-region globalization은 6회 accepted step으로 residual을 `1.553959e4 N`까지 낮추고, 3회차 이후 relative increment gate는 통과한다. Accepted-history secant subspace는 residual-only 후보 `1.550832e4 N`을 찾지만 relative increment `1.80e-03`으로 closure 후보가 아니며, gate-limited/finer alpha 후보를 6회 final state로 승격해 residual을 `1.552967e4 N`까지 낮춘다. 이어 matrix-free finite-difference consistent-Jacobian subspace는 최근 basis `1`과 `2` 후보를 pass마다 함께 평가하고 full residual/increment gate를 통과한 최선 후보를 8회 승격한다. Selected basis sequence는 `[2,1,2,2,2,2,2,1]`이며, 8회 모두 `1e-5` relative-improvement floor를 넘긴 뒤 `max_promotions_exhausted`로 멈추고, residual을 `1.551637e4 N`까지 더 낮추며 relative increment `9.499e-05`를 유지한다. 따라서 AI repair, surrogate, 최적화 정책은 fixed-point/remap frontier나 residual-only 후보를 물리 residual closure로 승격하면 안 되며, consistent residual Jacobian 또는 arc-length/trust-region globalization이 direct residual gate와 increment gate를 동시에 통과할 때까지 review-required/unsupported로 남겨야 한다.

AI routing should now prefer generalized energy/coarse-space quality and multilevel interface coupling over scalar RHS/load-partition tuning, while continuing to block any solver-closure claim until full residual gates pass.
- GENEO basis-width sensitivity is now measured. Sweeping local generalized modes-per-DOF `[1,2,3,4]` improves the focused shell route to `22.673608427637085 N` at `3` modes/DOF, while `4` modes/DOF worsens to `22.83171465230876 N`. AI routing should therefore stop treating coarse-space widening as automatically beneficial; it should recommend mode selection, multilevel transfer, and interface Schur/DD structure with replay gates.
- GENEO local mode ordering by RHS is now counter-evidence. At `3` modes/DOF, the default low-eigen ordering stays best at `22.673608427637085 N`; `rhs_projection` and `rhs_energy_score` worsen to `23.197418644884337 N` and `23.182806652657437 N`. AI routing should not recommend simple load-projection mode ranking as closure-grade basis learning; it should move to multilevel transfer, interface Schur/DD structure, or a richer learned selector validated by replay.
- GENEO scalar damping/pass tuning is now also bounded. A focused weight/pass sweep improves only to `22.6733256606884 N`, less than `1e-3 N` below the prior best, and larger repeated corrections worsen sharply. AI routing should stop proposing more one-level damping/pass search and instead require a different operator level: multilevel transfer, interface Schur complement, or domain decomposition coupling.
- A minimal Schur-after-GENEO cycle was tested and is counter-evidence. Reapplying the same translation/rotation Schur approximation after GENEO coarse correction worsens every nonzero cycle row; the best remains the zero-cycle `22.6733256606884 N` frontier. AI routing should not suggest merely cycling the same approximate Schur and coarse blocks; it should require a stronger complement/DD operator or different transfer hierarchy.
- One-hop harmonic GENEO transfer has now been tested as the smallest operator-space change after scalar tuning saturated. `interface_edge_geneo_harmonic_restricted` extends selected interface GENEO modes through adjacent CSR rows with Jacobi diagonal scaling and keeps the correction on the ROCm/HIP Galerkin path. The best focused shell row uses extension weight `0.5`, records `26189` extension DOFs, and lowers residual replay from `22.6733256606884 N` to `22.593705557868677 N`; full weight `1.0` worsens to `23.170971964048427 N`. AI routing should treat this as evidence that transfer quality matters, not as closure; G9 still requires true multilevel/interface Schur/DD quality before the solver gap can close.
- Harmonic transfer depth is now tested and bounded. With extension weight fixed at `0.5`, depth `1` remains best at `22.593705557868677 N`; depth `2` worsens to `22.899531912544845 N` and depth `3` worsens to `22.993533780768566 N` despite larger extension supports. AI routing should avoid recommending deeper single-level Jacobi extension as closure and should instead require stabilized multilevel transfer, coarse solve quality, or interface/domain-decomposition coupling.
- QR stabilization of the harmonic GENEO coarse basis is now measured. It gives a very small one-hop improvement (`22.593705557868677 -> 22.5936722466134 N`) and a similarly tiny two-hop improvement while leaving two-hop worse than one-hop. AI routing should keep QR as a conditioning guard, but not treat it as a substitute for a true multilevel/interface Schur/DD operator.

Switching the harmonic GENEO + QR route from load-vector restriction to current-residual restriction worsens residual replay from `22.5936722466134 N` to `23.26506328336595 N`. AI routing should preserve selective load-target restriction and move to stronger multilevel/interface Schur or domain-decomposition coupling instead of residual-space projection inside the same single-level basis.

The best row remains the disabled smoother at `22.5936722466134 N`; the best nonzero smoother row is worse at `22.81499423648211 N`. Multiplicative update improves the nonzero local solve only slightly (`22.814994235978013 -> 22.814749673790256 N`) and still loses to the disabled smoother baseline `22.5936722466134 N`. AI routing should therefore stop blaming stale additive residual application and instead target interface/coarse coupling quality or a materially stronger DD/AMG operator.
- Post-interface coarse rebalance was tested as the next coupling check and is negative. Reapplying the same harmonic GENEO coarse correction after multiplicative interface-pair DD worsens to roughly `32.01 N`, while no rebalance keeps `22.5936722466134 N`. AI routing should not classify same-coarse-after-local-DD cycling as multilevel closure; the next useful route needs a different stable interface/coarse hierarchy or AMG/DD operator.
- A small signed rebalance-weight sweep confirms this is not just a bad damping constant. With multiplicative interface-pair smoothing fixed at `0.001`, no rebalance remains best at `22.814749673790256 N`; signed rebalance weights `[-0.01,-0.001,0.001,0.01]` all worsen, with the best rebalance row `23.015726093014894 N`. AI routing should therefore change the coarse/interface space or hierarchy, not continue tuning the same post-DD coarse direction.
- Residual-region-guided local DOF filtering is also counter-evidence. The all-DOF harmonic GENEO + QR row keeps `100%` of top-64 residual rows on translation DOFs, but filtering the coarse basis to translations only reduces coarse columns from `362` to `167` and worsens residual replay from `22.5936722466134 N` to `23.128330475949674 N`; rotation-only uses `195` columns and worsens to `23.246764238877844 N`. AI routing should treat translation residual concentration as a diagnostic, not as proof that a translation-only coarse space is sufficient. The next G9 proposal should alter transfer/coarse hierarchy or Schur/DD operator quality, not local DOF filtering.
- Galerkin-energy orthogonalization of the same harmonic GENEO basis is now tested and does not improve. The `energy` mode diagonalizes `B^T A B` and rescales all `362` columns, but residual replay worsens slightly from the QR row `22.5936722466134 N` to `22.59379546634976 N`. AI routing should therefore not recommend reconditioning the same single-level subspace as the next closure route; it should require a genuinely different hierarchy/interface Schur/DD operator.
